# weekly-pulse

The fastest possible status read. Intended to take the reader 2 minutes to absorb. Prepended (newest-first) to `client-memory.md` as a durable weekly record.

## When to use

Trigger phrases: "weekly pulse", "this week's status", "pulse <client>", "give me the 2-minute", "what's new for <client>".

## Trin 0 — Kontekst (skal altid køres først)

Læs `${CLAUDE_PLUGIN_ROOT}/CLAUDE.md` før noget andet. Den indeholder write-gate-reglerne, kilde-attribution-formatet, sprog-reglerne, og hvordan Drive bruges.

## Trin 1 — Verificér Drive

Bekræft at du kan se Inbound-rod-mappen i Drive (ID `${user_config.inbound_root_folder_id}`, default `17JwnWKToZSJUSCURjS9PzzBeqe6_gPfi`). Hvis ikke, sig: "Jeg kan ikke nå Inbound's Drive-mappe. Tjek at du er logget ind på Drive i Cowork." og stop.

## Inputs (from Drive)

- `05-data/metrics-weekly.csv` (or equivalent) — last 4 weeks
- `04-memory/client-memory.md` — prior context, last pulse
- `06-decisions/*.md` — decisions whose measurement window is active

## What to produce

Exactly three sections, short:

### What moved
The 3-5 metrics that changed meaningfully this week vs last week. Include direction and size. Skip metrics that were flat.

### What's at risk
1-3 things that need attention this week or next. Specific. Each with a one-line rationale.

### What the client will likely ask about
1-2 sentences. Based on what's running, what they flagged last meeting, and what's imminent on their calendar (if memory tells us).

## Rules

- 2 minutes to read. Cut ruthlessly.
- Numbers where numbers exist; qualitative only if there's no alternative.
- Do NOT restate the whole monthly story. This is a *delta* view.
- Do NOT recommend actions — that's proactivity-scan's job. Pulse is observation + forecast.
- Always end output with `## Kilder` listing every Drive file you read. Format per `${CLAUDE_PLUGIN_ROOT}/CLAUDE.md` § Source attribution.
- **Drive write-back is human-in-the-loop. Never write to `client-memory.md` without explicit user approval.** Memory is newest-first: new entries are **prepended to the top** of the file (see `${CLAUDE_PLUGIN_ROOT}/CLAUDE.md` § Memory ordering). After producing the pulse, render the proposed entry block and ask the user to confirm before writing. Use this exact pattern:

  1. Show the pulse to the user
  2. Render the proposed entry block in a fenced code block, prefixed with: *"Proposed entry to prepend at the top of `04-memory/client-memory.md` — confirm to write, edit to revise, or say skip."*
  3. Wait for explicit approval (`yes` / `approve` / `confirm` / `write it`). Do NOT write on silence, on a thumbs-up, or on a continuation prompt. Edits are also approval — apply edits then write.
  4. Only after approval, perform the Drive write — **prepend** the entry at the top of the file, above all prior entries. Never append to the bottom.
  5. Confirm the write back to the user with the file path and confirm the entry now sits at the top.

  Entry format (when approved). The date in the header must be today's date in `YYYY-MM-DD`:

  ```
  ## YYYY-MM-DD — Weekly pulse (auto-generated by /weekly-pulse)

  **Period covered:** <start> to <end>

  **What moved:**
  - <bullet>
  - <bullet>

  **Watch items:**
  - <bullet>

  **What Sara will likely ask about next:** <one line>
  ```

The user-confirmed prepend is the write-back step. Without it, the pulse evaporates — but a write without confirmation is worse than no write at all.

## Example output shape

```
# Nordkap Friluft — weekly pulse (2026-04-20)

**Period:** 2026-04-12 → 2026-04-20

## What moved
- Tent CVR held at 0.95% for the 4th week running — category page flip 3 weeks in

- Email open rate drifted back up to 19.8% (from 19.5% low) — single send, not a trend yet
- Hav & Fjeld still showing on pack queries — no change from last week's pull

## At risk
- Re-permissioning campaign still waiting on Sara — greenlight pending since April 8
- Blended ROAS 3.2x four weeks running, Meta still the weak side

## What Sara will likely ask
Whether category page flip is showing in behaviour metrics yet. Honest answer: 3 weeks in, directionally holding, too early for CVR attribution.

```
